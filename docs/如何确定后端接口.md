### **第二步：查看接口定义**

打开 `user.py` 文件，找到登录接口：

```python
# src/backend/agentchat/api/v1/user.py

from fastapi import APIRouter, Body

router = APIRouter(tags=["User"])

@router.post('/user/login', response_model=UnifiedResponseModel)
async def login(
    user_name: str = Body(description='用户名'),      # ← 请求参数1
    user_password: str = Body(description='用户密码')  # ← 请求参数2
):
    """
    用户登录接口
    
    参数：
    - user_name: 用户名
    - user_password: 用户密码
    
    返回：
    - user_id: 用户ID
    - access_token: JWT Token
    """
    # 1. 查询数据库
    db_user = UserDao.get_user_by_username(user_name)
    
    # 2. 验证密码
    if not UserService.verify_password(user_password, db_user.user_password):
        return {"status_code": 401, "status_message": "用户名或密码错误"}
    
    # 3. 生成 JWT Token
    access_token = create_jwt_token(db_user)
    
    # 4. 返回响应
    return {
        "status_code": 200,
        "status_message": "success",
        "data": {
            "user_id": db_user.user_id,
            "access_token": access_token
        }
    }
```

**关键信息**：
- **路径**：`/user/login`
- **方法**：`POST`
- **请求参数**：`user_name`（用户名）、`user_password`（密码）
- **返回数据**：`user_id`（用户ID）、`access_token`（JWT Token）

### **第三步：查看路由前缀**

打开 `router.py` 文件：

```python
# src/backend/agentchat/api/router.py

from fastapi import APIRouter

router = APIRouter(prefix="/api/v1")  # ← 所有接口都有这个前缀

router.include_router(user.router)     # 注册 user.py 的路由
```

**完整路径**：`/api/v1` + `/user/login` = `/api/v1/user/login`

## 🎓 实战案例：如何对接一个新接口

假设你要实现**获取智能体列表**功能：

### **第一步：查看 API 文档**

访问 `http://localhost:7860/docs`，找到：

```
GET /api/v1/agent
获取智能体列表

Query Parameters:
  user_id: string (必填)

Response:
{
  "status_code": 200,
  "data": [
    {
      "id": "string",
      "name": "string",
      "description": "string"
    }
  ]
}
```

---

### **第二步：在前端封装 API**

创建或编辑 `src/frontend/src/apis/agent.ts`：

```typescript
import { request } from '../utils/request'

// 获取智能体列表
export const getAgentListAPI = (userId: string) => {
  return request({
    url: '/api/v1/agent',      // ← 从 API 文档获取
    method: 'GET',              // ← 从 API 文档获取
    params: {                   // ← GET 请求用 params
      user_id: userId           // ← 从 API 文档获取参数名
    }
  })
}
```

---

### **第三步：在 Vue 组件中调用**

```vue
<script setup lang="ts">
import { ref, onMounted } from 'vue'
import { getAgentListAPI } from '../../apis/agent'

const agentList = ref([])

onMounted(async () => {
  try {
    const response = await getAgentListAPI('user_123')
    if (response.data.status_code === 200) {
      agentList.value = response.data.data
    }
  } catch (error) {
    console.error('获取智能体列表失败:', error)
  }
})
</script>

<template>
  <div>
    <div v-for="agent in agentList" :key="agent.id">
      {{ agent.name }}
    </div>
  </div>
</template>
```

## 📋 接口对接检查清单

在对接接口时，确保以下信息一致：

- [ ] **路径**：前端 `url` = 后端 `@router.xxx('/path')`
- [ ] **方法**：前端 `method` = 后端 `@router.post/get/put/delete`
- [ ] **参数名**：前端 `data/params` 的字段名 = 后端 `Body/Query` 的参数名
- [ ] **参数类型**：前端传的数据类型 = 后端接收的数据类型
- [ ] **返回格式**：前端解析的字段 = 后端返回的字段

## 🔍 常见问题

### **Q1：前端调用接口报 404 Not Found**

**原因**：接口路径不正确

**解决**：
1. 检查前端 `url` 是否正确
2. 访问 `http://localhost:7860/docs` 确认接口路径
3. 检查后端是否启动

---

### **Q2：前端调用接口报 422 Unprocessable Entity**

**原因**：请求参数不正确

**解决**：
1. 检查参数名是否正确（大小写敏感）
2. 检查参数类型是否正确（string/number/boolean）
3. 检查必填参数是否都传了

---

### **Q3：前端调用接口报 401 Unauthorized**

**原因**：JWT Token 无效或过期

**解决**：
1. 检查 localStorage 中是否有 token
2. 重新登录获取新 token
3. 检查请求头是否正确添加了 `Authorization: Bearer <token>`

---

### **Q4：GET 请求用 params 还是 data？**

**答案**：
- **GET 请求**：用 `params`（拼接到 URL 后面）
- **POST/PUT 请求**：用 `data`（放在请求体中）

```typescript
// GET 请求
request({
  url: '/api/v1/agent',
  method: 'GET',
  params: { user_id: '123' }  // ← 用 params
})
// 实际请求：GET /api/v1/agent?user_id=123

// POST 请求
request({
  url: '/api/v1/user/login',
  method: 'POST',
  data: {                      // ← 用 data
    user_name: 'admin',
    user_password: '123456'
  }
})
```

## 🎯 总结

### **确定后端接口的 3 种方法**

1. **查看 API 文档**（最推荐）
   - 访问 `http://localhost:7860/docs`
   - 可以直接测试接口

2. **查看后端代码**
   - 打开 `src/backend/agentchat/api/v1/*.py`
   - 查看 `@router.xxx()` 装饰器

3. **查看前端 API 封装**
   - 打开 `src/frontend/src/apis/*.ts`
   - 查看已有的接口调用

---

### **对接新接口的步骤**

```
1. 查看 API 文档，确定接口路径、方法、参数
   ↓
2. 在前端 apis/ 目录创建 API 封装函数
   ↓
3. 在 Vue 组件中调用 API 函数
   ↓
4. 处理响应数据，更新界面
```

---

## 💡 实用技巧

1. **使用 API 文档测试接口**：在写前端代码前，先在文档中测试接口是否正常
2. **统一错误处理**：在 Axios 拦截器中统一处理 401、500 等错误
3. **TypeScript 类型定义**：为 API 请求和响应定义 TypeScript 接口
4. **API 分模块管理**：按功能模块创建不同的 API 文件（auth.ts、agent.ts 等）
