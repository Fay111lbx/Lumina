## 🔪 切分方法详解

### **方法1：纯文本切分（TXT、HTML、JSON、CSV、Excel）**

**代码位置**：`src/backend/agentchat/services/rag/doc_parser/text.py`

#### **切分策略：按行切分 + 重叠**

```python
class TextParser:
    def __init__(self):
        self.chunk_size = 500      # 每块最大 500 字符
        self.overlap_size = 100    # 重叠 100 字符
    
    async def split_text_into_chunks_by_lines(self, text):
        """按换行符切割文本，保留重叠部分"""
        lines = text.splitlines()  # 按行分割
        chunks = []
        current_chunk = []
        current_length = 0
        
        for line in lines:
            line_length = len(line)
            
            # 如果加上新行超过 chunk_size，保存当前 chunk
            if current_length + line_length > self.chunk_size:
                if current_chunk:
                    chunk = "\n".join(current_chunk)
                    chunks.append(chunk)
                    
                    # 保留重叠部分：从末尾截取 overlap_size 个字符
                    overlap = chunk[-self.overlap_size:]
                    current_chunk = [overlap] if overlap else []
                    current_length = len(overlap)
            
            # 添加行到当前 chunk
            current_chunk.append(line)
            current_length += line_length
        
        # 处理最后一个 chunk
        if current_chunk:
            chunk = "\n".join(current_chunk)
            chunks.append(chunk)
        
        return chunks
```

### **方法2：Markdown 切分（智能切分）**

**代码位置**：`src/backend/agentchat/services/rag/doc_parser/markdown.py`

#### **切分策略：按标题层级 + 段落 + 重叠**

```python
class MarkdownParser:
    def __init__(self):
        self.min_chunk_size = 256   # 最小块大小
        self.max_chunk_size = 512   # 最大块大小
        self.overlap_size = 128     # 重叠大小
```

#### **核心特点**

1. **保留标题层级**：每个 chunk 都包含完整的标题路径
2. **智能切分**：优先在段落边界切分，避免切断句子
3. **保护链接**：不会切断 Markdown 链接和图片
4. **动态调整**：根据内容长度自动调整切分位置

#### **Markdown 切分的核心算法**

```python
async def parse_markdown_headers(self, text):
    """解析 Markdown 标题结构，按标题切分"""
    
    current_headers = {1: '', 2: '', 3: '', 4: '', 5: ''}  # 1-5级标题
    chunks = []
    current_text = []
    
    for line in text.split('\n'):
        # 检查是否是标题行
        if line.startswith('#'):
            # 遇到标题前，先处理积累的文本
            if current_text:
                header_path = ' > '.join([h for h in current_headers.values() if h])
                chunks.extend(await self.split_text_with_headers(
                    '\n'.join(current_text), 
                    header_path
                ))
                current_text = []
            
            # 更新标题层级
            level = len(line.split()[0])  # 统计 # 的数量
            header_text = line.lstrip('#').strip()
            current_headers[level] = header_text
            
            # 清除子标题
            for l in range(level + 1, 6):
                current_headers[l] = ''
        else:
            # 收集普通文本
            current_text.append(line)
    
    return chunks
```

class PDFParser:
    async def parse_into_chunks(self, file_id, file_path, knowledge_id):
        # 1. 将 PDF 转换为 Markdown
        markdown_file = await self.convert_markdown(file_path)

        )
    
    async def convert_markdown(self, file_path: str):

        md_text = pymupdf4llm.to_markdown(
            doc=file_path,

        )

        # 重写 Markdown 中的图片链接

        return markdown_file

## 🎯 切分方法对比

| 文档类型 | 切分方法 | chunk_size | overlap_size | 特点 |
|---------|---------|-----------|-------------|------|
| **TXT/HTML/JSON** | 按行切分 | 500 | 100 | 简单快速 |

## 🔧 可选功能：Chunk 摘要生成

**配置**：
```yaml

```

**代码位置**：`src/backend/agentchat/services/rag/parser.py`

```python
async def generate_summary(cls, chunk: ChunkModel):
    """为每个 chunk 生成摘要（约100字）"""
    
    prompt = f"""
    你是一个专业的摘要生成助手，请根据以下要求为文本生成一段摘要：
    
    ## 需要总结的文本：
    {chunk.content}
    
    ## 要求：
    1. 摘要字数控制在 100 字左右。
    2. 摘要中仅包含文字和字母，不得出现链接或其他特殊符号。
    3. 只输出摘要部分，不准输出 `以下是文本的摘要` 等字段
    """
    
    response = await llm.ainvoke(prompt)
    chunk.summary = response.content
    
    return chunk
```

## 🎓 总结

### **OmniAgent 的文档切分方法**

| 参数 | 默认值 | 说明 |
|------|--------|------|
| **chunk_size** | 500 | 每个文档块的最大字符数 |
| **overlap_size** | 100 | 文档块之间的重叠字符数 |
| **min_chunk_size** | 256 | Markdown 的最小块大小 |
| **max_chunk_size** | 512 | Markdown 的最大块大小 |

### **核心特点**

1. **智能切分**：
   - 纯文本：按行切分，保留重叠
   - Markdown：按标题层级切分，保留结构
   - PDF：转 Markdown 后智能切分

2. **上下文保留**：
   - 重叠部分保证上下文连贯
   - Markdown 保留完整标题路径

3. **格式保护**：
   - 不切断 Markdown 链接
   - 不切断句子（优先在句子边界切分）

4. **多格式支持**：
   - 文本、Markdown、PDF、Word、PPT、图片、Excel

### **切分流程**

```
上传文档 → 识别格式 → 提取内容 → 切分成块 → 生成摘要（可选） → 向量化 → 存储
```

### **优化建议**

1. **如果文档内容很长**：增加 chunk_size 到 800-1000
2. **如果需要更多上下文**：增加 overlap_size 到 150-200
3. **如果需要更精确的检索**：启用 enable_summary
4. **如果是技术文档**：使用 Markdown 格式（保留结构）

---

**简单说**：这个项目的文档切分方法是**智能的、保留上下文的、多格式支持的**，能够根据不同文档类型选择最合适的切分策略！
