## 1. 项目架构理解

### 1.1 整体架构

```
┌─────────────────────────────────────────────────────────┐
│                      用户浏览器                          │
│                   (Vue 3 前端界面)                       │
└────────────────────┬────────────────────────────────────┘
                     │ HTTP/WebSocket
                     ↓
┌─────────────────────────────────────────────────────────┐
│                   FastAPI 后端服务                       │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐             │
│  │ API 路由 │→ │ 业务逻辑 │→ │ Agent 层 │             │
│  └──────────┘  └──────────┘  └──────────┘             │
└────────┬────────────┬────────────┬─────────────────────┘
         │            │            │
         ↓            ↓            ↓
    ┌────────┐  ┌─────────┐  ┌──────────┐
    │ MySQL  │  │  Redis  │  │  MinIO   │
    │ 数据库 │  │  缓存   │  │ 文件存储 │
    └────────┘  └─────────┘  └──────────┘
         │
         ↓
    ┌──────────────────────────────────┐
    │      大语言模型 (通义千问)        │
    │   + 工具调用 (天气、搜索等)       │
    └──────────────────────────────────┘
```

### 1.2 为什么这样设计？

**前后端分离的好处：**
- 前端专注于用户交互和界面展示
- 后端专注于业务逻辑和数据处理
- 可以独立开发、测试、部署
- 前端可以轻松替换（Web、移动端、桌面端）

**为什么需要这么多服务？**
- **MySQL**: 存储结构化数据（用户、对话、消息等）
- **Redis**: 缓存热点数据，提高响应速度
- **MinIO**: 存储文件（图片、文档、头像等）
- **向量数据库**: 存储知识库的向量表示，用于语义搜索

## 2. 技术栈选型原因

### 2.1 后端：为什么选 FastAPI？

```python
# FastAPI 的优势示例
from fastapi import FastAPI

app = FastAPI()

@app.get("/api/chat")
async def chat(message: str):
    # 1. 自动参数验证
    # 2. 自动生成 API 文档
    # 3. 原生支持异步
    # 4. 性能接近 Go/Node.js
    return {"response": "Hello"}
```

**对比其他框架：**
- **Django**: 功能全面但较重，适合传统 Web 应用
- **Flask**: 轻量但需要很多扩展，异步支持不好
- **FastAPI**: 现代化、高性能、自动文档、类型提示

### 2.2 前端：为什么选 Vue 3？

```vue
<!-- Vue 3 的组合式 API -->
<script setup>
import { ref, onMounted } from 'vue'

const messages = ref([])

onMounted(async () => {
  // 简洁的响应式数据管理
  messages.value = await fetchMessages()
})
</script>
```

**对比其他框架：**
- **React**: 生态最大，但学习曲线陡峭
- **Vue 3**: 易学易用，性能优秀，中文文档好
- **Angular**: 企业级，但过于复杂

### 2.3 为什么选 LangChain？

LangChain 是一个 LLM 应用开发框架，提供了：
- **统一的模型接口**: 可以轻松切换不同的大模型
- **工具调用封装**: 简化 Function Calling 的实现
- **Agent 框架**: 提供了 ReAct、Plan-and-Execute 等模式
- **记忆管理**: 对话历史、长期记忆等

## 3. 核心功能实现原理

### 3.1 对话功能是如何实现的？

**完整流程：**

```
用户输入 "今天北京天气怎么样？"
    ↓
前端发送请求到后端 /api/v1/chat
    ↓
后端接收请求，提取消息内容
    ↓
查询数据库，获取对话历史
    ↓
构建 Prompt（系统提示词 + 历史对话 + 用户输入）
    ↓
调用大模型 API（通义千问）
    ↓
模型判断需要调用天气工具
    ↓
后端执行天气查询工具（调用高德地图 API）
    ↓
将工具结果返回给模型
    ↓
模型生成最终回答
    ↓
保存对话到数据库
    ↓
流式返回给前端显示
```

**关键代码位置：**
- 前端发送请求: `src/frontend/src/apis/chat.ts`
- 后端接收处理: `src/backend/agentchat/api/v1/chat.py`
- Agent 逻辑: `src/backend/agentchat/core/agent.py`
- 工具定义: `src/backend/agentchat/tools/weather.py`

### 3.2 工具调用（Function Calling）原理

**什么是工具调用？**

让大模型能够调用外部函数/API 来获取实时信息或执行操作。

**示例：天气查询工具**

```python
# src/backend/agentchat/tools/weather.py

from langchain.tools import BaseTool

class WeatherTool(BaseTool):
    name = "get_weather"
    description = "查询指定城市的天气信息。输入城市名称，返回天气数据。"
    
    def _run(self, city: str) -> str:
        """执行天气查询"""
        # 1. 调用高德地图 API
        api_key = settings.tools.weather.api_key
        url = f"https://restapi.amap.com/v3/weather/weatherInfo"
        params = {
            "key": api_key,
            "city": city,
            "extensions": "base"
        }
        
        # 2. 发送请求
        response = requests.get(url, params=params)
        data = response.json()
        
        # 3. 解析结果
        if data["status"] == "1":
            weather = data["lives"][0]
            return f"{city}天气：{weather['weather']}，温度{weather['temperature']}℃"
        else:
            return "查询失败"
```

**模型如何知道要调用工具？**

```python
# 在 Prompt 中告诉模型有哪些工具可用
tools = [
    {
        "name": "get_weather",
        "description": "查询城市天气",
        "parameters": {
            "type": "object",
            "properties": {
                "city": {"type": "string", "description": "城市名称"}
            }
        }
    }
]

# 模型会返回类似这样的结果
{
    "tool_calls": [
        {
            "name": "get_weather",
            "arguments": {"city": "北京"}
        }
    ]
}
```

### 3.3 RAG（知识库检索）原理

**什么是 RAG？**

Retrieval-Augmented Generation（检索增强生成）：让模型能够访问外部知识库。

**完整流程：**

```
1. 文档上传
   用户上传 PDF/Word 文档
   ↓
   文档解析（提取文本）
   ↓
   文本分块（Chunking）
   ↓
   向量化（Embedding）
   ↓
   存入向量数据库

2. 查询时
   用户提问 "公司的请假制度是什么？"
   ↓
   问题向量化
   ↓
   在向量数据库中搜索相似文档
   ↓
   取出 Top-K 最相关的文档片段
   ↓
   将文档片段 + 用户问题一起发给模型
   ↓
   模型基于文档内容生成回答
```

**关键代码：**

```python
# 文档向量化
from langchain.embeddings import OpenAIEmbeddings
from langchain.vectorstores import Chroma

# 1. 加载文档
documents = load_documents("company_policy.pdf")

# 2. 分块
chunks = split_documents(documents, chunk_size=500)

# 3. 向量化并存储
embeddings = OpenAIEmbeddings()
vectorstore = Chroma.from_documents(chunks, embeddings)

# 4. 查询
query = "请假制度"
relevant_docs = vectorstore.similarity_search(query, k=3)
```

### 3.4 多 Agent 协作原理

**什么是多 Agent？**

将复杂任务分解给多个专门的 Agent 处理。

**示例场景：**

```
用户: "帮我写一篇关于 AI 的文章，并配图"

主 Agent (协调者)
    ↓
    ├─→ 搜索 Agent: 搜索 AI 相关资料
    ├─→ 写作 Agent: 根据资料写文章
    └─→ 绘图 Agent: 根据文章内容生成配图
    
最终整合结果返回给用户
```

## 4. 数据流转过程

### 4.1 用户发送消息的完整流程

**前端部分：**

```typescript
// src/frontend/src/apis/chat.ts

export async function sendMessage(message: string) {
  // 1. 构建请求数据
  const data = {
    message: message,
    dialog_id: currentDialogId,
    agent_id: selectedAgentId
  }
  
  // 2. 发送 POST 请求
  const response = await fetch('/api/v1/chat/stream', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(data)
  })
  
  // 3. 处理流式响应
  const reader = response.body.getReader()
  while (true) {
    const { done, value } = await reader.read()
    if (done) break
    
    // 4. 逐字显示
    const text = new TextDecoder().decode(value)
    appendToMessage(text)
  }
}
```

**后端部分：**

```python
# src/backend/agentchat/api/v1/chat.py

@router.post("/chat/stream")
async def chat_stream(request: ChatRequest):
    # 1. 验证用户身份
    user = get_current_user(request)
    
    # 2. 获取对话历史
    history = await get_dialog_history(request.dialog_id)
    
    # 3. 构建 Agent
    agent = create_agent(
        agent_id=request.agent_id,
        tools=get_agent_tools(request.agent_id),
        memory=history
    )
    
    # 4. 流式生成回复
    async for chunk in agent.astream(request.message):
        yield chunk
    
    # 5. 保存到数据库
    await save_message(request.dialog_id, request.message, response)
```

## 5. 如何添加新功能

### 5.1 添加一个新的工具（以快递查询为例）

**第一步：创建工具类**

```python
# src/backend/agentchat/tools/delivery.py

from langchain.tools import BaseTool
import requests

class DeliveryTool(BaseTool):
    name = "query_delivery"
    description = "查询快递物流信息。输入快递单号，返回物流详情。"
    
    def _run(self, tracking_number: str) -> str:
        """查询快递"""
        api_key = settings.tools.delivery.api_key
        url = settings.tools.delivery.endpoint
        
        headers = {"Authorization": f"APPCODE {api_key}"}
        params = {"no": tracking_number}
        
        response = requests.get(url, headers=headers, params=params)
        data = response.json()
        
        if data["status"] == "200":
            return self._format_result(data["result"])
        else:
            return "查询失败"
    
    def _format_result(self, result):
        """格式化输出"""
        info = []
        for item in result["list"]:
            info.append(f"{item['time']}: {item['status']}")
        return "\n".join(info)
```

## 6. 常见问题排查

### 6.1 后端启动失败

**问题：数据库连接失败**
```
sqlalchemy.exc.OperationalError: (pymysql.err.OperationalError) 
(2003, "Can't connect to MySQL server")
```

**排查步骤：**
1. 检查 MySQL 是否启动：`mysql -u root -p`
2. 检查配置文件中的数据库地址、端口、密码
3. 检查数据库是否已创建：`SHOW DATABASES;`

**问题：Redis 连接失败**
```
redis.exceptions.ConnectionError: Error connecting to Redis
```

**排查步骤：**
1. 检查 Redis 是否启动：`redis-cli ping`
2. 检查配置文件中的 Redis 地址

### 6.2 前端无法连接后端

**问题：跨域错误**
```
Access to fetch at 'http://localhost:7860' from origin 'http://localhost:5173' 
has been blocked by CORS policy
```

**解决方法：**

后端已经配置了 CORS，检查 `main.py`:
```python
app.add_middleware(
    CORSMiddleware,
    allow_origins=['*'],  # 允许所有来源
    allow_credentials=False,
    allow_methods=['*'],
    allow_headers=['*'],
)
```

### 6.3 大模型调用失败

**问题：API Key 无效**
```
Error: Invalid API key
```

**排查步骤：**
1. 检查 API Key 是否正确复制（没有多余空格）
2. 检查 API Key 是否已激活
3. 检查账户余额是否充足

**问题：模型返回空内容**

可能原因：
1. Prompt 设计不合理
2. 模型参数设置问题（temperature、max_tokens）
3. 网络超时
